# UX Decisions Game

Un juego de decisiones de producto sobre consecuencias, no respuestas correctas.

## ¿Qué es esto?

**UX Decisions Game** es un experimento interactivo donde el jugador asume el rol de una persona que toma decisiones de producto.

No se trata de diseñar interfaces.  
No se trata de aprender teoría de UX.

Se trata de entender cómo decisiones cotidianas —muchas veces razonables y tentadoras— van moldeando, con el tiempo, la experiencia de usuario, el soporte y la salud del negocio.

No hay decisiones correctas.  
Hay trade-offs.

---

## ¿Qué explora este proyecto?

- Cómo decisiones de corto plazo generan deuda de UX a largo plazo  
- Cómo optimizar una métrica suele afectar negativamente a otras  
- Cómo la fricción se acumula de forma silenciosa antes de hacerse visible  
- Cómo muchas malas experiencias nacen de decisiones razonables tomadas sin una mirada sistémica  

UX no aparece de forma explícita.  
Emerje a través de las consecuencias.

---

## ¿Cómo funciona?

- El juego está organizado en niveles  
- Cada nivel contiene cartas con situaciones reales de producto  
- En cada carta, el jugador debe tomar una decisión  
- Cada decisión:
  - Consume tiempo (días, sprints, semanas)
  - Impacta en múltiples métricas del sistema
- Las consecuencias se acumulan entre decisiones  
- El estado del sistema no se reinicia entre cartas  

Una mecánica central es la **deuda de UX**:
- Se acumula lentamente  
- No duele de inmediato  
- Termina afectando al resto de las métricas  

Mantener todo “en verde” es intencionalmente difícil.

---

## ¿Qué no es este proyecto?

- No es un tutorial de UX  
- No es un simulador de interfaces  
- No es un framework  
- No es un juego de productividad  

Es una experiencia sobre criterio, tensión y consecuencias no intencionales.

---

## Tecnología

Este proyecto está construido intencionalmente con:

- HTML  
- CSS  
- JavaScript vanilla  

Sin frameworks.  
Sin build tools.  
Sin dependencias.

La simplicidad técnica es parte del experimento.
